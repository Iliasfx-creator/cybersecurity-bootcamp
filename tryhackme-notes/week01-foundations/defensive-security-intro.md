Concepts

Defensive security focuses on prevention, detection, investigation, response
and recovery.

Important sources of evidence include:

Authentication events
Authorization decisions
Application logs
Network telemetry
Endpoint activity
Audit trails
Alerts and incident timelines
Connection with Product Security
Secure design must provide useful telemetry. Privileged actions need actor,
tenant, target, result and correlation identifiers so defensive teams can
investigate abuse.
Preventive controls and detective controls complement each other.
Lesson learned
A control is incomplete if failure cannot be detected or investigated.
Evidence handling
No answers, flags, target addresses or credentials are stored.
