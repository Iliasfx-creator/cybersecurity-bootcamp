## Core concepts

- Confidentiality protects information from unauthorized disclosure.
- Integrity protects information from unauthorized or accidental change.
- Availability keeps systems and data usable when needed.
- Authentication establishes an identity.
- Authorization decides which actions that identity may perform.
- Accountability connects actions to reliable evidence.
- Least privilege gives only the permissions required.
- Defense in depth uses multiple complementary controls.

## Risk thinking

- Security is not only a list of technical vulnerabilities. Risk depends on the asset, threat, weakness, likelihood, impact and effectiveness of controls.
- Controls should be selected according to the root cause and verified through positive and negative tests.

## Connection with the bootcamp

- These principles apply directly to session security, access control, API authorization, audit logging and threat modeling.
